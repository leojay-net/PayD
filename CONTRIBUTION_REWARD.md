# Contribution Reward (Bounty) Program

PayD encourages open‑source contributions by offering bounties on high‑priority issues. This document outlines the program, eligibility, and claim process.

---

## 🎯 When Is a Bounty Offered?
- Issues marked with the **`bounty`** label in the GitHub repository.
- The issue description includes a **reward amount** and the **currency** (XLM or a stable asset).
- Bounties are typically reserved for:
  - Critical bug fixes that impact payroll reliability.
  - Core feature implementations that advance the product roadmap.
  - Significant documentation enhancements for bounty‑tagged issues.

## ✅ Eligibility Criteria
- The contribution must be **original** and **fully functional**.
- All code must pass the repository's CI pipeline (`npm run test:docs`).
- The PR must be **merged** into the default branch.
- Contributors must follow the project's **code of conduct**.

## 📥 How to Claim a Bounty
1. **Find a bounty‑tagged issue** on the GitHub Issues board.
2. **Fork the repository** and create a branch for your work.
3. **Implement the solution** and ensure all tests pass.
4. **Open a Pull Request** referencing the issue (e.g., `Fixes #123`).
5. **Comment on the issue** after the PR is merged, stating that you are claiming the bounty and provide the PR link.
6. A maintainer will verify the work and, if approved, will initiate the reward transfer.

## 💰 Reward Distribution
- Rewards are paid in **Stellar Lumens (XLM)** or the specified stable asset.
- Payments are processed via the project's Stellar wallet within **7 business days** of PR merge.
- The maintainer will request the contributor’s Stellar address if not already provided.

## 📜 Transparency & Reporting
- All bounty payments are recorded on the Stellar ledger, providing a transparent audit trail.
- A summary of paid bounties is maintained in the `BOUNTY_LOG.md` file in the repository root.
