# Security Policy

PayD takes security seriously. We appreciate the efforts of security researchers and the community in responsibly disclosing vulnerabilities.

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

To report a vulnerability, use GitHub's private security advisory feature:

👉 [**Submit a Vulnerability Report**](https://github.com/Gildado/PayD/security/advisories/new)

This ensures your report is handled privately and confidentially.

### What to Include

When submitting a report, please provide as much of the following as possible:

- A clear description of the vulnerability
- The affected component(s) and version(s)
- Steps to reproduce the issue
- Proof-of-concept or exploit code (if available)
- The potential impact of the vulnerability
- Any suggested mitigations or fixes

## Response Timeline

| Stage                  | Target Time    |
| ---------------------- | -------------- |
| Initial acknowledgment | Within 48 hours |
| Status update          | Within 7 days  |
| Resolution / patch     | Within 90 days |

We will keep you informed throughout the process and notify you when the vulnerability has been resolved.

## Supported Versions

We actively maintain and patch the latest release. Older versions may not receive security updates.

| Version | Supported |
| ------- | --------- |
| Latest  | ✅ Yes    |
| Older   | ❌ No     |

## Responsible Disclosure

We ask that you:

- Give us reasonable time to investigate and remediate before public disclosure
- Make a good-faith effort to avoid privacy violations and service disruptions
- Do not access or modify data belonging to other users

In return, we commit to:

- Acknowledging your report promptly
- Working with you to understand and resolve the issue
- Crediting you in our security acknowledgments (if you wish)
- Not pursuing legal action against researchers acting in good faith

## Acknowledgments

We thank all security researchers who have contributed to making PayD more secure. Recognized contributors will be listed here with their permission.

## Security.txt

A machine-readable `security.txt` file is available at:

```
/.well-known/security.txt
```

This file follows the [RFC 9116](https://www.rfc-editor.org/rfc/rfc9116) standard.

## Contact

For general security questions (not vulnerability reports), you can open a [GitHub Discussion](https://github.com/Gildado/PayD/discussions).
